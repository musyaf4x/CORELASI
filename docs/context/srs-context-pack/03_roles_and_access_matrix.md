# 03 Roles And Access Matrix

Catatan penting: `Admin`, `Guru`, dan `Siswa` adalah role utama pada model `User.role`. `Guru Pengampu`, `Guru Piket`, dan `Wali Kelas` bukan role utama terpisah; ketiganya adalah penugasan/scope di bawah role `Guru`.

| Role / Penugasan | Fitur yang Bisa Diakses | Fitur yang Dilarang | Guard/Permission yang Mengatur | Bukti Codebase |
| ---------------- | ----------------------- | ------------------- | ------------------------------ | -------------- |
| Admin | Dashboard admin, CRUD user, akademik, jadwal pembelajaran, jadwal piket, absensi, monitoring jurnal, laporan absensi/nilai/operasional, profil, ubah password. | Tidak boleh menghapus akun dirinya sendiri. | Frontend `RoleLayout allowedRoles={["admin"]}`; backend permission `IsAdminOrReadOnly`, `IsAdminRole`; `UserViewSet.destroy` self-delete guard. | `frontend/src/app/router.tsx`; `accounts/views.py`; `academic/views.py`; `schedules/views.py`. |
| Guru | Dashboard guru, jadwal, absensi sesuai scope, materi, tugas, jurnal, grading, kelas, laporan kelas, profil, ubah password. | Tidak boleh mengakses route admin/siswa; tidak boleh mengelola data akademik master; tidak boleh menilai tugas milik guru lain; tidak boleh membuat materi/tugas di kelas-mapel yang tidak diajar. | Frontend `RoleLayout allowedRoles={["guru"]}`; backend `can_teach_subject`, `teacher_class_ids`, `IsOwnerOrAdmin`; queryset filtering di learning/attendance/journals/reports. | `frontend/src/app/router.tsx`; `shared/access.py`; `learning/views.py`; `attendance/views.py`; `journals/views.py`. |
| Siswa | Dashboard siswa, jadwal kelasnya, riwayat absensi sendiri, materi/tugas terpublikasi kelasnya, submit tugas, nilai, profil, ubah password. | Tidak boleh mengakses route admin/guru; tidak boleh membuat materi/tugas; tidak boleh menilai; tidak boleh melihat data siswa lain. | Frontend `RoleLayout allowedRoles={["siswa"]}`; backend queryset `user.role == "siswa"` filter by user/kelas; submission create hanya role siswa. | `frontend/src/app/router.tsx`; `learning/views.py`: `SubmissionViewSet.create`; `attendance/views.py`: `get_queryset`; `schedules/views.py`: `get_queryset`. |
| Guru Pengampu | Mengelola materi/tugas untuk kelas-mapel yang diajar, mengisi absensi kelas-mapel yang diajar, menilai submission tugasnya, melihat siswa pada kelas yang diajar. | Tidak boleh mengelola kelas/mapel yang tidak diajar. | `can_teach_subject(user, kelas_id, mapel_id)` berdasarkan `JadwalPelajaran` semester aktif. | `shared/access.py`: `can_teach_subject`; `schedules/models.py`: `JadwalPelajaran`; `learning/views.py`; `attendance/views.py`. |
| Guru Piket | Memiliki akses koreksi/verifikasi absensi dan scope absensi lebih luas ketika bertugas piket. | Tidak otomatis menjadi admin; tetap tidak boleh mengelola user/akademik master. | `is_duty_teacher(user)` memakai flag `is_piket_today` atau `JadwalPiket` hari ini pada semester aktif. | `shared/access.py`: `is_duty_teacher`; `schedules/models.py`: `JadwalPiket`; `attendance/views.py`: `PermintaanKoreksiViewSet.verify`. |
| Wali Kelas | Mengakses scope kelas perwalian dan laporan/perwalian di UI guru. | Bukan role utama; tidak otomatis boleh mengelola semua data sekolah. | `teacher_class_ids` union kelas yang diajar dan `Kelas.wali_kelas`; frontend `AssignmentGuard requiredAssignment="isWaliKelas"`. | `shared/access.py`: `teacher_class_ids`; `academic/models.py`: `Kelas.wali_kelas`; `frontend/src/components/shared/AssignmentGuard.tsx`. |
| Superuser/operator | Secara model Django dapat login sebagai user role admin dengan `is_staff/is_superuser` untuk operasi maintenance; tidak tampil di quick login. | Tidak boleh menghapus akun dirinya sendiri; tidak boleh dimasukkan ke quick login showcase. | Django flags `is_staff/is_superuser`; `UserViewSet.destroy` self-delete guard; showcase allowlist via settings. | `accounts/models.py`: custom `User`; `accounts/views.py`: `UserViewSet.destroy`, `ShowcaseLoginView`; `config/settings.py`: `SHOWCASE_ACCOUNT_EMAILS`; handoff production state. |

Rekomendasi untuk editor SRS:

1. Pertahankan `Admin`, `Guru`, dan `Siswa` sebagai role utama.
2. Tuliskan `Guru Pengampu`, `Guru Piket`, dan `Wali Kelas` sebagai penugasan akademik, bukan role login terpisah.
3. Jangan memasukkan superuser/operator sebagai aktor akademik utama; cukup catat sebagai akun operasional bila diperlukan.
